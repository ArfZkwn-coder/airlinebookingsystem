### GitHub Protocol
When pushing code:
1. Verify git initialization (`git status`)
2. Check remote (`git remote -v`), ask for URL if missing
3. Stage & commit (`git add .` + `git commit -m "..."`)
4. Push (`git push origin <branch>`)
5. Confirm with commit hash and branch
6. Log event to `references/memory.md`

**Rules:** No force-push without approval, always confirm branch, surface merge conflicts clearly, never resolve silently.
